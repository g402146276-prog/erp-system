# ERP系统部署到腾讯云 + 宝塔面板 完整说明

---

## 第一部分：准备工作

### 1.1 登录腾讯云后台

打开浏览器访问 https://console.cloud.tencent.com/cvm ，找到你的服务器列表。

### 1.2 放行端口（安全组）

左侧菜单 → **安全组** → 找到你服务器绑定的安全组 → **修改规则**

添加入站规则：

| 协议 | 端口 | 来源 | 说明 |
|------|------|------|------|
| TCP | 22 | 0.0.0.0/0 | SSH（如果开宝塔远程登录需要）|
| TCP | 80 | 0.0.0.0/0 | HTTP访问 |
| TCP | 443 | 0.0.0.0/0 | HTTPS（配域名后）|
| TCP | 8888 | 0.0.0.0/0 | 宝塔面板（临时）|
| TCP | 8000 | 0.0.0.0/0 | API测试（临时后可关）|

> 8888和8000用完后建议关掉，只保留80和443。

### 1.3 获取服务器信息

在腾讯云CVM列表里找：

- **公网IP**（如 123.123.123.123）
- **操作系统**（建议 CentOS 7+ 或 Ubuntu 20+）
- **用户名/密码**（或密钥对）

## 第二部分：安装宝塔面板

### 2.1 登录服务器

**Windows电脑** 打开 PowerShell 或 CMD，输入：

```bash
ssh root@你的服务器IP
```

> 如果提示没有ssh命令，先安装 OpenSSH，或者用 **宝塔远程工具**（腾讯云后台有网页版登录）。
>
> 也可以用 **Xshell**、**Putty** 等工具。

### 2.2 使用腾讯云"宝塔一键部署"

**最快的方法**（推荐）：

腾讯云后台 → 服务器列表 → 点击你的服务器 → **更多** → **运维** → **镜像/部署** → 选 **宝塔面板** 一键部署。

> 如果没有这个选项，用下面的方法2手动安装。

### 2.3 手动安装宝塔（备用）

登录服务器后，执行命令（CentOS 系统）：

```bash
yum install -y wget && wget -O install.sh https://download.bt.cn/install/install_6.0.sh && sh install.sh ed8484bec
```

Ubuntu 系统：

```bash
wget -O install.sh https://download.bt.cn/install/install-ubuntu_6.0.sh && bash install.sh ed8484bec
```

安装完成后会显示：

```
Bt-Panel: http://你的IP:8888/xxxxx
username: xxxxxx
password: xxxxxx
```

> **一定要保存好这个地址、用户名和密码！**

如果忘了，登录服务器后执行 `bt default` 查看。

### 2.4 登录宝塔面板

浏览器访问：`http://你的服务器IP:8888/xxxxx`

输入用户名密码登录。

登录后弹窗选 **同意协议**，然后选安装推荐环境：

- **LNMP 推荐**（Nginx + MySQL + PHP）
- 不需要PHP和MySQL的话，可以只装Nginx

## 第三部分：上传项目代码

### 3.1 在你本地电脑打包代码

打开 **本地的 PowerShell**，执行：

```bash
cd C:\Users\Administrator\Desktop
tar -czf erp-system.tar.gz erp-system
```

> 如果提示 tar 不存在，装一个 7-Zip 或 WinRAR，把 erp-system 文件夹压缩成 .zip 或 .tar.gz。

### 3.2 通过宝塔上传

宝塔面板 → 左侧 **文件** → 进入 `/www/wwwroot/` → 点 **上传** → 选择 `erp-system.tar.gz`

等待上传完成。

### 3.3 解压

右键上传的文件 → **解压**

解压后路径应该是：`/www/wwwroot/erp-system/`

---

## 第四部分：配置Python后端

### 4.1 打开宝塔软件商店

左侧 **软件商店** → 搜索 **Python 项目部署器** → **安装**

### 4.2 添加Python项目

打开 **Python项目** → **添加Python项目**：

| 设置项 | 填写内容 |
|--------|----------|
| 项目名称 | `erp-backend` |
| 监听端口 | `8000` |
| 项目路径 | `/www/wwwroot/erp-system/backend` |
| Python版本 | 选 3.10 或 3.11（没有就点旁边安装） |
| 框架 | 点 **自定义** |
| 启动方式 | **uvicorn** |
| 启动文件 | `app.main:app` |
| 命令行 | 自动生成不用改 |
| 项目虚拟环境 | **勾选** 自动创建 |

点 **确认**。

### 4.3 安装依赖

**方法1：宝塔自动安装**

Python项目列表 → 点 **依赖** → 等它自动 `pip install -r requirements.txt`

**方法2：手动安装**

文件 → 进入 `/www/wwwroot/erp-system/backend/` → 顶部 **终端** → 执行：

```bash
source /www/wwwroot/erp-system/backend/venv/bin/activate
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 4.4 检查项目是否有requirements.txt

如果没有，手动进入 `backend` 目录→终端，执行：

```bash
source venv/bin/activate
pip install fastapi uvicorn sqlalchemy python-jose[cryptography] passlib[bcrypt] python-multipart aiofiles
pip freeze > requirements.txt
```

### 4.5 启动项目

Python项目列表 → 点 **启动**（或重启）

查看日志确认是否启动成功，日志应该显示：

```
Uvicorn running on http://0.0.0.0:8000
```

**验证**：浏览器访问 `http://你的服务器IP:8000/health`，应该返回 `{"status": "healthy"}`

## 第五部分：编译前端

### 5.1 进入前端目录

宝塔 → 文件 → 进入 `/www/wwwroot/erp-system/frontend/`

### 5.2 打开终端

点顶部 **终端**（或 SSH 登录后执行）

```bash
cd /www/wwwroot/erp-system/frontend
```

### 5.3 安装依赖并编译

```bash
npm install
npm run build
```

编译完成后会出现 `dist` 文件夹，里面有 `index.html` 和 `assets` 目录。

> 如果提示 `npm: command not found`，先装 Node.js：
> 宝塔 → 软件商店 → 搜 **Node.js版本管理器** → 安装 → 选一个版本（18或20均可）

## 第六部分：配置Nginx网站

### 6.1 添加网站

宝塔 → 左侧 **网站** → **添加站点**：

| 设置项 | 填写内容 |
|--------|----------|
| 域名 | `你的服务器IP`（如果有域名填域名） |
| 根目录 | `/www/wwwroot/erp-system/frontend/dist` |
| FTP | 不创建 |
| 数据库 | 不创建 |

点 **提交**。

### 6.2 设置反向代理

点刚创建的网站 → **设置** → **反向代理** → **添加**：

| 设置项 | 填写内容 |
|--------|----------|
| 名称 | `api` |
| 目标URL | `http://127.0.0.1:8000` |
| 内容替换 | 不填 |
| 发送域名 | `$host` |

### 6.3 修改Nginx配置文件（重要）

网站设置 → **配置文件**

找到 `server { ... }` 块里面的 `location /`，在这之后添加：

```nginx
location /api/ {
    proxy_pass http://127.0.0.1:8000/api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}

location /health {
    proxy_pass http://127.0.0.1:8000/health;
}
```

**完整配置示例**（可以全选替换）：

```nginx
server {
    listen 80;
    server_name 你的服务器IP;
    
    # 前端静态文件
    root /www/wwwroot/erp-system/frontend/dist;
    index index.html;

    # API反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /health {
        proxy_pass http://127.0.0.1:8000/health;
    }

    # SPA路由：所有非文件请求返回 index.html
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

点 **保存**。

### 6.4 重启Nginx

网站设置 → 点 **重启**（或左侧软件商店→Nginx→重载配置）

## 第七部分：访问测试

### 7.1 打开浏览器

访问 `http://你的服务器IP`

应该能看到登录页面！

### 7.2 如果打不开

检查顺序：

1. **Nginx是否启动**：宝塔 → 网站 → 看状态是不是绿色
2. **Python后端是否启动**：宝塔 → Python项目 → 看状态是不是运行中
3. **端口是否放行**：腾讯云安全组有没有放行80端口？
4. **检查日志**：点Python项目的 **日志** 看有没有报错

### 7.3 常见问题排查

**问题：页面打开了但接口报错**
```bash
# 服务器上直接测试API
curl http://127.0.0.1:8000/health
```
如果返回正常，说明后端OK，问题在Nginx配置。

**问题：登录后一直跳转**
检查前端API地址配置，需要改成服务器的API路径。

编辑 `/www/wwwroot/erp-system/frontend/src/api/index.js`，确认 baseURL：

```javascript
// 生产环境不需要baseURL（通过Nginx同域名访问）
// 开发环境用 http://localhost:8000
const API_BASE = ''  // 留空即可
```

改完后需要重新 `npm run build`。

## 第八部分：配置域名和HTTPS（可选）

### 8.1 绑定域名

如果你有域名，DNS解析到服务器IP后：

1. 宝塔 → 网站 → 点网站名 → 修改域名 → 填你的域名
2. 在 **域名管理** 里把域名添加进去

### 8.2 免费SSL证书

宝塔 → 网站 → SSL → **Let's Encrypt** → 勾选域名 → **申请**

申请成功后打开 **强制HTTPS**。

### 更新代码

从本地重新打包上传，或者用git：

1. 宝塔 → 文件 → 进入项目目录

3. Python项目 → 重启

- Nginx → 设置 → 网站日志

宝塔 → 数据库 → 找到 erp 的数据库 → 点 **备份**

```bash
# 进入桌面
cd C:\Users\Administrator\Desktop

# 打包（不包含node_modules和venv）
tar --exclude=erp-system/frontend/node_modules --exclude=erp-system/backend/venv -czf erp-system.tar.gz erp-system
```
